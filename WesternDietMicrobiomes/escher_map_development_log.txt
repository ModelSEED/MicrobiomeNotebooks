KEY CHALLENGES & SOLUTIONS
================================================================================

1. BLANK HTML RENDERING
   Problem: Generated HTML files appeared blank in Firefox and Brave browsers
   Solution:
   - Used local escher.min.js copied from installed package
   - Proper HTML template with full-screen container styling
   - DOMContentLoaded event handler
   - zoom_extent_canvas() call after delay

2. BUILDER.MAP UNDEFINED FOR MULTI-REACTION MAPS
   Problem: Single reaction maps worked, but maps with 2+ reactions failed
            with builder.map being undefined

   Investigation:
   - Tested incrementally: 1 reaction worked, 2+ failed
   - Model map with 115 reactions worked
   - Compared structure between working and failing maps

   ROOT CAUSE DISCOVERED:
   Segment IDs must be GLOBALLY UNIQUE across all reactions, not just unique
   within each reaction.

   Evidence from model map:
   - Reaction 0: segment IDs = ['0', '1', '2', '3', '4']...
   - Reaction 1: segment IDs = ['21', '22', '23', '24', '25']...
   - Reaction 2: segment IDs = ['35', '36', '37', '38', '39']...

   Fix: Added global segment_id_counter that increments across all reactions

3. METABOLITE LAYOUT ORGANIZATION
   Requirement: Position metabolites based on their role across all ASVs

   Implementation:
   - Only consumed (reactants): LEFT side of ASV nodes
   - Only produced (products): RIGHT side of ASV nodes
   - Mixed (both consumed and produced): CENTER near ASV nodes

   Added categorize_metabolites() function to analyze flux patterns

================================================================================
ESCHER MAP STRUCTURE REQUIREMENTS
================================================================================

1. JSON format: Array with two elements
   [0]: Header object (map_name, map_id, schema, etc.)
   [1]: Data object (reactions, nodes, text_labels, canvas)

2. Node types:
   - "multimarker": Connection point for metabolites
   - "midmarker": Center of reaction
   - "metabolite": Metabolite node with bigg_id, name, coordinates

3. Segment structure:
   - from_node_id, to_node_id: String node IDs
   - b1, b2: Bezier control points (can be null for straight lines)
   - CRITICAL: Segment IDs must be globally unique across entire map

4. Reaction structure:
   - name, bigg_id, reversibility
   - label_x, label_y: Position of reaction label
   - metabolites: Array of {bigg_id, coefficient}
   - segments: Object with unique segment IDs as keys
